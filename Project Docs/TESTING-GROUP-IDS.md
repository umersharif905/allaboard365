## Testing Scenarios

### Test 1: Automatic Group ID Generation (ARM)

**Steps:**
1. Create a new group (or use existing group)
2. Assign ARM vendor products to the group:
   - One product with "Master" in the name
   - One product with "CoPay" or "Co-Pay" in the name
   - One product with "HSA" in the name
3. Save the group

**Expected Results:**
- First group should get: 90500 (Master), 90501 (CoPay), 90502 (HSA)
- Second group should get: 90505 (Master), 90506 (CoPay), 90507 (HSA)
- Each subsequent group increments by 5

**Verify:**
```sql
SELECT 
    v.VendorName,
    gp.GroupProductId,
    p.Name as ProductName,
    vgi.VendorGroupId,
    vgi.ProductType,
    vgi.IsAutoGenerated,
    g.Name as GroupName
FROM oe.GroupProductVendorGroupIds vgi
INNER JOIN oe.GroupProducts gp ON vgi.GroupProductId = gp.GroupProductId
INNER JOIN oe.Products p ON gp.ProductId = p.ProductId
INNER JOIN oe.Vendors v ON vgi.VendorId = v.VendorId
INNER JOIN oe.Groups g ON gp.GroupId = g.GroupId
WHERE v.VendorName LIKE '%ARM%'
  AND vgi.IsActive = 1
ORDER BY g.Name, vgi.VendorGroupId
```

### Test 2: Manual Group ID Creation

**Steps:**
1. Get a GroupProductId and VendorId (not ARM)
2. Use API endpoint:
   ```bash
   POST /api/vendor-group-ids
   Authorization: Bearer <token>
   Content-Type: application/json
   
   {
     "groupProductId": "<group-product-id>",
     "vendorId": "<vendor-id>",
     "vendorGroupId": "CUSTOM001",
     "productType": "Default"
   }
   ```

**Expected Results:**
- Group ID created successfully
- `IsAutoGenerated` = 0 (false)
- Can retrieve via GET endpoint

**Verify:**
```sql
SELECT 
    vgi.VendorGroupId,
    vgi.IsAutoGenerated,
    v.VendorName,
    p.Name as ProductName
FROM oe.GroupProductVendorGroupIds vgi
INNER JOIN oe.GroupProducts gp ON vgi.GroupProductId = gp.GroupProductId
INNER JOIN oe.Products p ON gp.ProductId = p.ProductId
INNER JOIN oe.Vendors v ON vgi.VendorId = v.VendorId
WHERE vgi.VendorGroupId = 'CUSTOM001'
```

### Test 3: Update Group ID

**Steps:**
1. Use PUT endpoint:
   ```bash
   PUT /api/vendor-group-ids/group-product/:groupProductId/vendor/:vendorId
   Authorization: Bearer <token>
   Content-Type: application/json
   
   {
     "vendorGroupId": "CUSTOM002"
   }
   ```

**Expected Results:**
- Group ID updated successfully
- ModifiedDate and ModifiedBy set

### Test 4: Get Group IDs

**Steps:**
1. Use GET endpoint:
   ```bash
   GET /api/vendor-group-ids/group/:groupId/vendor/:vendorId
   Authorization: Bearer <token>
   ```

**Expected Results:**
- Returns all Group IDs for the group and vendor
- Includes ProductType, ProductName, etc.

### Test 5: ARM Export View

**Steps:**
1. Ensure group has ARM products with Group IDs assigned
2. Run ARM export or query the view:
   ```sql
   SELECT TOP 10
       [Group Number],
       [Last Name],
       [First Name],
       [Employee Or Dependent]
   FROM oe.v_ARM_Export_Data
   WHERE [Group Number] LIKE '90%'
   ORDER BY [Group Number]
   ```

**Expected Results:**
- `[Group Number]` should show generated Group IDs (90500, 90501, etc.)
- Not showing Group Name anymore

### Test 6: Duplicate Prevention

**Steps:**
1. Try to create a Group ID that already exists for the same vendor
2. Should fail with error message

**Expected Results:**
- Error: "Group ID '90500' is already in use for this vendor"

## API Endpoints Reference

- `GET /api/vendor-group-ids/group/:groupId/vendor/:vendorId` - Get all Group IDs for a group
- `GET /api/vendor-group-ids/group-product/:groupProductId/vendor/:vendorId` - Get specific Group ID
- `POST /api/vendor-group-ids` - Create manual Group ID
- `PUT /api/vendor-group-ids/group-product/:groupProductId/vendor/:vendorId` - Update Group ID
- `DELETE /api/vendor-group-ids/group-product/:groupProductId/vendor/:vendorId` - Delete Group ID

## Troubleshooting

### Group IDs Not Generating
- Check ARM vendor has GroupIdPrefix = "90" and GroupIdSeedNumber = 500
- Check products have VendorId set
- Check backend logs for errors

### Duplicate Group ID Error
- Check unique index is working: `UQ_GroupProductVendorGroupIds_Vendor_VendorGroupId_Active`
- Verify no active duplicate records exist

### Export View Not Showing Group IDs
- Verify view was updated: Check `oe.v_ARM_Export_Data` view definition
- Check Group IDs exist for the products in the export
- Verify Group IDs are active (`IsActive = 1`)
